Display Inline vs. Inline Blocks in CSS

HTML elements are catergorized as either block-level elements or inline elements. Inline elements do not start on a new line, they displays on the same line to the other contents. While block elements always starts on a new line and takes up the full width available on that section. This blog will show you how the display: Inline and display: Inline-blocks properties in CSS changesthe layout of those elements.

Inline Elements:

Inline elements do not take up all the space of its container. It usually sits within other elements and only takes as much space as it’s content needs. And thus, you can have multiple Inline elements in the same line. However, you can not have a width and height set for an inline element. Examples of inline elements are <span> <em> <img> :

Block Elements:

The behavior of Block level element is that it takes up all the space of its rectangle anyway. Which means, it doesn’t let other elements use the empty space of its own rectangle, by default it takes the full width of the page. The block element forces a line break after itself. Examples of block elements are: <div> <p> <fig>:

Display: Properties

The display property specifies the display behaviour of the elements. By default, the display behaviour is defined by the tags of the content, as states above, there are inline elements and block elements. In the CSS file I have added the display: inline property, we can see that the the block element <div> has now displayed like an inline element. There is no line breaks between each contents and it only takes the space that is needed for the content itself. The display: inline property overrides the default setting and redefines the display behaviour of the elements.
In this case I have chagnged the CSS file to display: inline-block, we can see that the elements are still inline, but they are showing like a box. The basic difference between the two properties is that, display: inline completely ignores the width and hight set, display: inline-block does not ignore height and width and developer can set width and height for each elements.
